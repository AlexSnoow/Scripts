# PowerShell Backup Toolkit — Документация

## Обзор

Автономный скрипт резервного копирования на PowerShell 2.0 (совместим с Windows 7).
Использует только архиватор **RAR** для создания зашифрованных архивов с проверкой целостности.

### Основные возможности

- **Единый конвейер архивации** — унифицированная логика без дублирования кода
- **Три режима архивации**:
  - Обычный: весь источник в один архив
  - Индивидуальные файлы: каждый файл — отдельный архив
  - Индивидуальные папки: каждая подпапка — отдельный архив
- **Верификация архивов** — проверка содержимого архива против источника (включая файлы 0 байт)
- **Ротация файлов** — автоматическое удаление старых архивов по политикам хранения
- **Сетевые отчёты** — централизованный сбор XML/CSV отчётов через NetLogPath
- **Интеграция с Windows Event Log** — запись событий запуска/завершения/ошибок
- **Уведомления по SMTP** — отправка отчётов на почту
- **Проверка целостности конфигурации** — SHA256 хеширование XML и RAR.exe

## Быстрый старт

### Запуск скрипта

```powershell
# Обычный запуск
powershell.exe -executionpolicy RemoteSigned -file .\app\Backup-ps2-v4.ps1

# Тестовый режим (проверка без выполнения)
powershell.exe -executionpolicy RemoteSigned -file .\app\Backup-ps2-v4.ps1 -testmode

# Проверка линтером
powershell.exe -executionpolicy RemoteSigned -file .\app\PS_linter.ps1
```

### Запуск тестов

```powershell
Invoke-Pester .\app\tests\Backup.Tests.ps1
```

## Конфигурация

Скрипт использует XML-файл `Backup-Config-All.xml` для настройки заданий.

### Структура конфигурации

```xml
<BackupConfig>
  <General>...</General>     # Общие настройки
  <Paths>...</Paths>         # Пути к ресурсам
  <Recipients>...</Recipients> # Адреса получателей почты
  <Integrity>...</Integrity>   # Хеш-контроль файлов
  <Jobs>...</Jobs>           # Список заданий архивации
</BackupConfig>
```

Подробная документация: [Backup_Config_Reference.md](Backup_Config_Reference.md)

## Режимы архивации

### 1. Обычный режим (Normal)

Весь источник архивируется в один архив.

### 2. Индивидуальная файловая архивация

Каждый файл создаётся как отдельный архив.

### 3. Индивидуальная папочная архивация

Каждая подпапка создаётся как отдельный архив.

Подробнее: [Backup_info.md#режимы-работы](Backup_info.md#режимы-работы)

## Процесс работы скрипта

### Этап 0: Проверка XML

1. Проверка наличия `Backup-Config-All.xml`
2. Валидация SHA256 хеша конфигурации

### Этап 1: Проверка архиватора

1. Проверка наличия RAR.exe
2. Валидация SHA256 хеша RAR.exe

### Этап 2: Основной запуск

1. Инициализация логирования
2. Проверка конфигурации
3. Для каждого задания:
   - Подготовка элементов архивации
   - Архивация через RAR
   - Верификация архива
   - Копирование в RemoteDest
   - Ротация локальных и удалённых архивов
   - Удаление источника (если включено)
4. Сохранение сетевых отчётов

Подробнее: [Backup_info.md](Backup_info.md)

## Логирование и отчёты

### Локальные логи

Формат имени: `<PCName>_<JobName>_<timestamp>.log`

### Сетевые отчёты (NetLogPath)

| Тип | Формат имени                         | Описание                    |
| --- | ------------------------------------ | --------------------------- |
| XML | `<PCName>_<JobName>_<timestamp>.xml` | Детальный отчёт по заданию  |
| CSV | `<PCName>_<JobName>_<timestamp>.csv` | Список файлов архива        |
| XML | `<PCName>_summary.xml`               | Сводный статус всех заданий |

## Безопасность

1. **Проверка целостности**: XML конфигурации и RAR.exe проверяются по SHA256
2. **Верификация архивов**: Содержимое архива сравнивается с источником
3. **Отсутствие секретов**: Токены и ключи не хранятся в скрипте
4. **EventLog аудит**: Все события записываются в системный лог

## Поддержка и контакты

При возникновении проблем проверьте:
1. Логи в `LogPathRoot`
2. Сетевые отчёты в `NetLogPath`
3. Windows Event Log (Application)
4. Настройки SMTP для получения уведомлений
