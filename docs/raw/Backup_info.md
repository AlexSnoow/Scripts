# Информация о системе резервного копирования

## Назначение

Система автоматизированного резервного копирования файлов на базе PowerShell 2.0 с использованием архиватора RAR.

## Архитектура

### Компоненты

```
┌─────────────────────────────────────────────────────────────┐
│                    Backup-ps2-v4.ps1                        │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────────────┐  │
│  │   Проверка  │  │  Архивация  │  │    Верификация      │  │
│  │  XML/RAR    │──│    RAR      │──│   (сравнение)       │  │
│  └─────────────┘  └─────────────┘  └─────────────────────┘  │
│                            │                                │

│  ┌─────────────┐  ┌─────────────┐  ┌─────────────────────┐  │
│  │ Логирование │  │   Ротация   │  │  Сетевые отчёты     │  │
│  │   (LOG)     │  │   файлов    │  │  (XML/CSV)          │  │
```

### Поток данных

1. **Источники** (Source) → Файлы для архивации
2. **RAR архивация** → Создание `.rar` архивов
3. **Локальное хранилище** (LocalDest) → Первичное хранение
4. **Удалённое хранилище** (RemoteDest) → Резервное копирование
5. **Сетевые отчёты** (NetLogPath) → Централизованный сбор метрик

## Режимы работы

### 1. Обычный режим (Normal)

Архивация всего содержимого источника в один архив.

**Применение:** Логические группы файлов, ежедневные бэкапы баз данных.

```xml
<Job Name="JOB1">
  <Source>C:\Work\BackupAllXml\src\JOB1</Source>
  <ArchivePattern>{PCName}_{JobName}_{Date_Time}.rar</ArchivePattern>
</Job>
```

### 2. Индивидуальная файловая архивация

Каждый файл становится отдельным архивом.

**Применение:** Лог-файлы с суффиксами дат, файлы с разными статусами.

```xml
<Job Name="JOB14">
  <Source>C:\Work\BackupAllXml\src\JOB14</Source>
  <ArchiveIndividualFiles>true</ArchiveIndividualFiles>
  <FileFilter>fxserver.20*.log</FileFilter>
  <IndividualArchivePattern>{PCName}_{JobName}_{SourceFileName}.rar</IndividualArchivePattern>
</Job>
```

### 3. Индивидуальная папочная архивация

Каждая подпапка создаётся как отдельный архив.

**Применение:** Ежедневные каталоги с данными, разделённые по датам.

```xml
<Job Name="JOB6">
  <Source>C:\Work\BackupAllXml\src\JOB6</Source>
  <ArchiveIndividualFolders>true</ArchiveIndividualFolders>
  <ExcludeFolderPattern>today</ExcludeFolderPattern>
  <IndividualArchivePattern>{PCName}_{JobName}_{SourceFolderName}.rar</IndividualArchivePattern>
</Job>
```

## Политики хранения

### Ротация источников

Параметры `SourceDaysOld` и `SourceKeepCount` контролируют удаление исходных файлов:

- `SourceDaysOld=0, SourceKeepCount=0` — удалить все после верификации
- `SourceDaysOld=7, SourceKeepCount=3` — хранить 7 дней, но не менее 3 файлов

### Ротация локальных архивов

`LocalDestDaysOld` и `LocalDestKeepCount`:

- Хранение по возрасту (дни)
- Минимальное количество файлов для сохранения

### Ротация удалённых архивов

`RemoteDestDaysOld` и `RemoteDestKeepCount`:

- Долгосрочное хранение на сетевых ресурсах
- Автоматическое освобождение места

## Верификация

### Процесс проверки

1. Сканирование источника (Get-FileList)
2. Чтение содержимого архива (Get-FileArhListRar)
3. Сравнение списков файлов (Compare-FilesSourceArchive)
4. Проверка размеров (включая файлы 0 байт)

### Критерии успеха

- Все файлы источника присутствуют в архиве
- Размеры файлов совпадают
- Нет лишних файлов в архиве

## Отчётность

### Форматы отчётов

**XML (детальный):**
```xml
<BackupReport>
  <Host>HOME-PC</Host>
  <Job>JOB1</Job>
  <Status>Success</Status>
  <SourceFiles>150</SourceFiles>
  <ArchiveSizeMB>245.5</ArchiveSizeMB>
  <Verification>Passed</Verification>
</BackupReport>
```

**CSV (список файлов):**
```
RelativePath;Length;LastWriteTime
data/file1.dat;1024;2026-04-12 20:54:49
data/file2.dat;2048;2026-04-12 20:54:50
```

**Сводный (summary.xml):**
```xml
<HostSummary>
  <Host>HOME-PC</Host>
  <LastRun>2026-04-12T20:54:49</LastRun>
  <Job name="JOB1" status="Success"/>
  <Job name="JOB6" status="Error" error="Верификация провалена"/>
</HostSummary>
```

## Интеграции

### Windows Event Log

События записываются в `Application` лог:

- **Start** (3000) — начало выполнения
- **Success** (3001) — успешное завершение
- **Warning** (3002) — предупреждения
- **Error** (3003) — ошибки
- **End** (3004) — завершение работы

### SMTP уведомления

Отправка отчётов на email:

- Успешное завершение
- Ошибки выполнения
- Тестовые результаты

## Безопасность

### Проверка целостности

1. **XML конфигурация** — SHA256 хеш в скрипте
2. **RAR.exe** — SHA256 хеш в конфигурации
3. **Архивы** — внутренняя проверка RAR (-s параметр)

### Контроль доступа

- ExecutionPolicy: RemoteSigned
- Права на чтение источников
- Права на запись в назначения

## Обработка ошибок

### Критические ошибки (остановка скрипта)

- Непройдена проверка XML/RAR хеша
- Отсутствует архиватор RAR
- Ошибки инициализации логирования

### Ошибки задания (продолжение других заданий)

- Отсутствует источник
- Ошибки архивации
- Ошибки верификации

### Предупреждения (продолжение работы)

- Сетевые отчёты не сохранены
- SMTP отправка не удалась
- Пустые источники

## Мониторинг

### Рекомендации

1. **Ежедневно:** Проверка сводных отчётов в NetLogPath
2. **Еженедельно:** Анализ логинов на наличие ошибок
3. **Ежемесячно:** Проверка политик ротации и освобождения места

### Ключевые метрики

- Количество успешных/неудачных заданий
- Общий размер архивов
- Время выполнения бэкапов
- Объём освобождённого места
