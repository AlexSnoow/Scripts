# Roadmap — Universal Toolkit Scripts

Этот файл является основным планом развития проекта. Он объединяет цели из `constitution.md`, требования из `specify.md` и технический стек из `plan.md`.

## 🎯 Цель проекта
Создание надежного, мультиплатформенного инструментария для автоматизации файловых операций (Резервное копирование, Копирование, Синхронизация, Очистка, Мониторинг) с поддержкой PowerShell 2.0 (Windows 7+) и Bash 3.0+ (Linux/Solaris 10+).

---

## 🏗 Архитектурные принципы
- **Unified Pipeline:** Все скрипты строго следуют 5 этапам: *Preparation → Main Operation → Verification → Post-Operations → Reporting*.
- **Platform Compatibility:** Строгое соблюдение ограничений версий (PS 2.0 / Bash 3.0+).
- **Security:** Отсутствие секретов в коде/конфигов, обязательная проверка хешей.
- **KISS & DRY:** Максимальная простота и использование общих функций из `docs/patterns/COMMON_FUNCTIONS.md`.

---

## 🗺 Этапы разработки

### Этап 1: Фундамент (Foundation)
*Обеспечение базовой инфраструктуры и общих компонентов.*
- [ ] Реализация системы логирования (INFO, WARN, ERROR, SUCCESS, DEBUG).
- [ ] Разработка системы обработки ошибок (try-catch / set -euo pipefail).
- [ ] Создание шаблонов конфигурации (XML для PS, .conf для Bash).
- [ ] Реализация общих функций (Write-Log, Send-Email, Test-FileIntegrity).

### Этап 2: Основные модули (Core Modules)
*Разработка профильных скриптов для каждой платформы.*
- [ ] **Backup:** Архивация с поддержкой RAR, 7zip, tar.gz.
- [ ] **Copy:** Копирование с верификацией и поддержкой режимов (ByDate, IndividualFiles и т.д.).
- [ ] **Sync:** Синхронизация (Incremental, Mirror, Full).
- [ ] **Cleanup:** Ротация файлов по правилам `KeepCount` и `DaysOld`.
- [ ] **Monitor:** Проверка системных параметров (диски, пути).

### Этап 3: Тестирование и Верификация (QA)
*Обеспечение надежности и соответствия спецификации.*
- [ ] Написание Pester тестов для всех PowerShell скриптов.
- [ ] Написание Bash unit-тестов.
- [ ] Проверка всех модулей в режиме `-testmode`.
- [ ] Линтинг кода (PS_linter.ps1).

### Этап 4: Документация и сдача (Finalization)
- [ ] Обновление `docs/` по мере завершения модулей.
- [ ] Финализация `docs/PIPELINE_TEMPLATE.md`.
- [ ] Проверка соответствия всем пунктам `specify.md`.
