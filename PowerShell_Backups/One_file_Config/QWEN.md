# BackupJOB1 — Система резервного копирования PowerShell

## Обзор проекта

**BackupJOB1** — это модульная система резервного копирования на PowerShell для Windows. Скрипт автоматизирует создание RAR-архивов из указанных каталогов, проверяет их целостность, управляет ротацией файлов и отправляет уведомления по электронной почте.

### Основные возможности

- **Многозадачность**: Поддержка нескольких заданий резервного копирования с индивидуальными настройками
- **Целостность данных**: Проверка хешей SHA256 всех исполняемых файлов перед запуском
- **Верификация архивов**: Сравнение содержимого источника и архива (имя, размер, дата) с автоматической нормализацией путей
- **Ротация файлов**: Автоматическое удаление старых резервных копий по возрасту и количеству
- **Логирование**: Детальные логи в UTF-8 с уровнями важности (INFO, WARNING, ERROR, SUCCESS)
- **Уведомления**: Отправка результатов по email через SMTP
- **Event Log**: Запись событий в журнал приложений Windows

## Конфигурация

### Backup-Config.json

Основной файл конфигурации содержит:

#### Раздел `General`
| Параметр | Описание |
|----------|----------|
| `JobName` | Имя задания (например, "BackupJOB1") |
| `Domain` | Домен для email-уведомлений |
| `LogDaysOld` | Возраст удаления логов (дни) |
| `LogKeepCount` | Количество сохраняемых логов |
| `DefaultRarParameters` | Параметры RAR по умолчанию |

#### Раздел `Paths`
| Параметр | Описание |
|----------|----------|
| `LogPathRoot` | Путь к локальным логам |
| `NetLogPath` | Путь к сетевым логам |
| `RarPath` | Путь к RAR.exe |
| `Modules.*` | Имена файлов модулей |

#### Раздел `Recipients`
| Параметр | Описание |
|----------|----------|
| `AdminIS` | Email администратора ИС |
| `AdminOS` | Email администратора ОС |
| `AdminMail` | Дополнительный email |

#### Раздел `Integrity`
SHA256-хеши для проверки целостности:
- `RarExeHash` — хеш RAR.exe
- `ModulesHashes` — хеши всех модулей (.psm1)

#### Раздел `Jobs` (массив заданий)
Каждое задание содержит:
| Параметр | Описание |
|----------|----------|
| `Name` | Имя задания (JOB1, JOB2, ...) |
| `Source` | Путь к исходным данным |
| `ArchivePattern` | Шаблон имени архива ({PCName}, {Date}) |
| `LocalDest` | Локальный путь назначения |
| `RemoteDest` | Сетевой путь назначения |
| `LocalDestDaysOld` | Возраст удаления локальных копий |
| `LocalDestKeepCount` | Количество локальных копий |
| `RemoteDestDaysOld` | Возраст удаления сетевых копий |
| `RemoteDestKeepCount` | Количество сетевых копий |
| `ArhParameters` | Параметры RAR для задания |
| `ArhLog` | Включить логирование RAR (true/false) |

## Запуск

### Основной способ (с JSON-конфигурацией)

```powershell
# Из директории проекта
.\Backup-Main.ps1
```

### Альтернативный способ (без JSON)

```powershell
# Конфигурация встроена в скрипт
.\Backup-Main-noJson.ps1
```

### Требования к запуску

1. **Проверка хешей**: Перед запуском скрипт проверяет SHA256-хеши всех зависимых файлов
2. **Права доступа**: Требуются права на чтение источников и запись в назначения
3. **Event Log**: Для записи в журнал Windows источник должен быть зарегистрирован:

```powershell
# Регистрация источника событий (требуется один раз, от администратора)
if (-not [System.Diagnostics.EventLog]::SourceExists("BackupJOB1")) {
    New-EventLog -LogName Application -Source "BackupJOB1"
}
```

## Процесс резервного копирования

1. **Проверка целостности**: Сверка хешей всех модулей и RAR.exe
2. **Импорт модулей**: Загрузка проверенных модулей PowerShell
3. **Инициализация логов**: Создание лог-файла с временной меткой
4. **Цикл по заданиям**:
   - Анализ источника (подсчёт файлов и размера)
   - Создание RAR-архива
   - Проверка целостности архива (тест RAR)
   - **Верификация**: Сравнение содержимого архива с источником
   - Копирование в сетевое назначение (если доступно)
   - Ротация старых файлов
5. **Отправка отчёта**: Email с результатами выполнения

## Безопасность

### Проверка хешей SHA256

Перед запуском скрипт проверяет целостность:
- `Backup-Config.json` — файл конфигурации
- `rar.exe` — исполняемый файл архиватора
- Все модули `.psm1`

При несовпадении хеша запуск блокируется с записью в Event Log.

### Обновление хешей в конфигурации

При изменении любого файла обновите хеш в `Backup-Config.json`:

```powershell
# Получить хеш файла
(Get-FileHash -Path ".\Backup-Logger.psm1" -Algorithm SHA256).Hash
```

## Коды возврата RAR

| Код | Описание |
|-----|----------|
| 0 | Успешное выполнение |
| 1 | Незначительная ошибка |
| 2 | Критическая ошибка |
| 3 | Ошибка проверки целостности |
| 6 | Файл заблокирован или недоступен |
| 255 | Операция прервана пользователем |

```json

  "LocalDest": "D:\\testBackupsLocal\\JOB5\\",
```

```powershell
$Script:ConfigExpectedHash = "<новый хеш>"
```

## Диагностика

### Проверка конфигурации

```powershell
# Проверка существования источников
Test-Path "D:\testSRC\JOB1\"

# Проверка доступности RAR
& "C:\Work\rar.exe" v

# Просмотр логов
Get-Content ".\logs\*.log" -Tail 50
```

### Event Log Windows

```powershell
# Просмотр событий скрипта
Get-WinEvent -FilterHashtable @{LogName='Application'; ProviderName='BackupJOB1'}

# Проверка источника событий
[System.Diagnostics.EventLog]::SourceExists("BackupJOB1")
```

### Тестирование верификации

```powershell
# Получить список файлов источника
$src = Get-FileList -Path "D:\testSRC\JOB1\"

# Получить список файлов архива
$arh = Get-FileArhList -RarPath "C:\Work\rar.exe" -ArchivePath "D:\backup.rar"

# Сравнить
$result = Compare-FilesSourceArchive -SourceList $src -ArchiveList $arh
$result.Report
```

## Известные ограничения

- **Однопоточность**: Задания выполняются последовательно
- **Кодировка**: Для русских имён файлов в RAR используется ключ `-scuc`
- **Сетевые пути**: Требуется доступ к сетевым ресурсам на момент выполнения
- **Event Log**: Источник должен быть зарегистрирован заранее (требуются права администратора)
