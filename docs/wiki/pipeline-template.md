# 5-Stage Pipeline Template

Мета-шаблон для создания скриптов с единым конвейером на PowerShell (Windows) и Bash (Linux/Solaris). Текущий охват: **Backup, Copy**.

## Принципы (из [[../constitution]])
- PS 2.0 совместимость
- Bash Solaris совместимость
- KISS — каждый этап = одна функция с single responsibility
- Единый конвейер — нарушение порядка этапов запрещено
- DRY — Write-Log, Send-Email, Remove-OldFiles общие для всех скриптов
- YAGNI — только то, что в конфиге
- Обработка ошибок — try/catch с логированием, запрещён SilentContinue

## Этапы

### 1. Preparation
Сканирование источников, проверка файлов по маскам, загрузка конфигурации, валидация, инициализация окружения.

### 2. Processing
Главный этап: создание архивов (RAR/7zip/tar.gz), копирование каждого файла из Source в RemoteDest.

### 3. Verification
Проверка целостности результата:
- Backup: код возврата архиватора, тест архива, сравнение списков
- Copy: сравнение размеров исходного и скопированного файла

### 4. Post-Operations
Копирование в сетевое хранилище, ротация (DaysOld/KeepCount), удаление источников после верификации.

### 5. Reporting
XML/CSV отчёты, email-уведомления, лог итогов. Выход: exit 0 (успех) / exit 1 (ошибки).

## Схема потока
```
[Preparation] -> [Processing] -> [Verification] -> [Post-Operations] -> [Reporting]
```

## Расширение шаблона
Шаблон спроектирован для будущих модулей (Sync, Cleanup, Monitor) по единой схеме.
Примеры реализации для других типов операций.

См. [[common-functions]] для реализации общих функций.
